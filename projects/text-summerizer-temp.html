Text Summerizer

An end-to-end, production-ready **Text Summarization** system powered by modern MLOps practices. This project leverages a modular architecture with configurable pipelines, version control, and seamless deployment using Docker, GitHub Actions, and AWS (EC2 + ECR).

Overview

This project is an end-to-end text summarization system built with a full MLOps pipeline and production-grade deployment. The system is designed to summarize conversational texts using the PEGASUS model fine-tuned on the Samsum dataset, making it ideal for real-world applications like chat or email summarization. It demonstrates a complete machine learning lifecycle, from experimentation and model training to scalable deployment using FastAPI, Docker, GitHub Actions, and AWS (EC2 + ECR).

Implementation Details

Walk through key components: data preprocessing, model architecture, training regime, evaluation metrics, and any optimizations or trade-offs.

  • 1. Modular Code Architecture

    The src/text_summarizer/ directory is divided into core submodules:

    • components/: Data loader, model trainer, inference engine.
    • pipeline/: Orchestrates the ML flow.
    • entity/ and config/: Used to decouple configuration from logic.
  • 2. Configurable Pipeline

    Training and inference behavior is fully controlled via config.yaml and params.yaml, promoting reproducibility and flexibility.

  • 3. Training
    • Data is tokenized using the tokenizer from google/pegasus-samsum.
    • Model is fine-tuned with early stopping and evaluation on a held-out validation set.
    • ROUGE metrics are computed post-training.
  • 4. FastAPI Inference Server

    app.py exposes /predict endpoint:

    {
        "text": "Hi John, how are you doing today?"
      }

    Returns an abstractive summary in JSON format.

  • 5. Docker & CI/CD
    • Dockerfile includes all system dependencies for clean deployment.
    • GitHub Actions pushes images to AWS ECR after successful build/test.
    • EC2 (Ubuntu) instance pulls and runs the container on update.
    • Secrets like AWS_ACCESS_KEY_ID are stored securely in GitHub.

Results & Impact

  • Achieved high-quality abstractive summaries with ROUGE-1 F1 score ~45, indicating strong semantic overlap.
  • Summaries are context-aware and optimized for dialogue-based text—more relevant than generic summarizers.
  • Reduced text length by 60–70% while preserving the core message—ideal for customer support and chat automation use-cases.
  • MLOps automation reduced manual deployment time from hours to under 5 minutes.
  • Application is production-ready and scalable via EC2 auto-scaling or future integration with container orchestration tools like ECS or Kubernetes.
